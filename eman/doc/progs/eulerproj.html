eulerproj

This program reassigns Euler angles to a set of class averages by projection matching

eulerproj <classes> <refrences> [sigfilt] [frac=<num>/<denom>] [mask=<radius>] [imask=<radius>] [phase] [writeali] [verbose]

Parameters:


<classes>The file containing the class averages to be oriented
<refrences>Projections to be used as references
[sigfilt]Apply a real-space filter to use only intense data in the classification
[frac=<num>/<denom>]Operate only on a fraction of the raw images
[mask=<radius>]Circular mask
[imask=<radius>]Circular innner mask
[phase]Use phase error as a similarity criteria
[writeali]Write the aligned images over the original images
[verbose]More verbose output during classification

Usage:

eulerproj classes.img proj.img mask=42

Description

When raw images are classified, there may be classification errors. Sometimes the misclassified images may outnumber the correctly classified images in specific classes. This means the class average generated from these images may not have the correct Euler angles. Since the class average is much less noisy, the Euler angle assignment is much more accurate for class averages. This program is used to redetermine the Euler angles for lower noise class averages. It could potentially be used to assign Euler angles to individual particles as well, although this practice is not part of the standard reconstruction procedure in EMAN.
