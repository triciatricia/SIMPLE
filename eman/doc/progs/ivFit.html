ivFit

This program fits the matched paires of helices returned by COSEC

ivFit <input to-be-transformed IV file> <input ref IV file> <out IV file> [matched=<n1,n2,n3,..>] [step=<angle>] [int] [verbose]

Parameters:


<input to-be-transformed IV file>The input Open Inventor format file for the helices to be transformed to match the <ref IV>
<input ref IV file>The input Open Inventor format file containing the ref helices
<out IV file>The output Open Inventor format file with transformed helices matching the ref helices
[matched=<n1,n2,n3,..>]set the corresponding helices in <ref IV file> that match the helices in <input to-be-transformed IV file>, -1 for non-match
[step=<angle>]step size in degree for the rotation fit, default to 5
[int]output the intermediate results with only the matched helices into file: ivFitProbe.iv and ivFitRef.iv
[verbose]will output some intermediate information

Usage:

ivFit probe.iv ref.iv probe-out.iv step=4 matched=2,6,-1,3,8,5 verbose

Description

This program uses Open Inventor library, so it is only working on SGI at this moment.

This program works correctly only for the special Open Inventor file containing only cylinders (helices). The matched option parameters come from the COSEC search result output.

Visit web page Helixhunter and Foldhunter for more information.
